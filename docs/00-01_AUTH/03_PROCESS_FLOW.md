# Process Flow

## Login flow

1. Користувач вводить email і пароль
2. UI передає дані через GAS/API
3. API приймає запит
4. API перевіряє email
5. API перевіряє пароль/token
6. API перевіряє статус користувача
7. API визначає роль користувача
8. API повертає результат авторизації

## Permission flow

1. API отримує `user_id`, `module`, `action`
2. API перевіряє роль користувача
3. API перевіряє дозвіл для ролі
4. API дозволяє або блокує дію
