## Clearance Policy Implemented

Before each new run:

- clears all output blocks
- sets run status to `RUNNING...`
- writes fresh request metadata (`request_id`, time)

On API error:

- engineering result blocks are kept empty
- only error block is written

On transport error:

- engineering result blocks are kept empty
- only transport error block is written

No fallback local calculations are performed.

## Request Envelope Implemented

Request payload generated by GAS:

- `request_id` UUID v4
- `client_type = GAS_DEMO`
- `mode = MODULE_01_DEMO`
- `product_type = KZO`
- `demo_id = MODULE_01_KZO_DEMO_001`
- default `requested_output_blocks`
- `operator_context.source = GOOGLE_SHEETS`
- `operator_context.sheet_name = MODULE_01_DEMO`

Forbidden flags are not included.

## Response/Header Validation Implemented

GAS validates:

- HTTP response code (2xx expected)
- JSON parsing
- header `X-EDS-Power-Mode = DEMO`
- envelope `status`
- `metadata.request_id` equals sent request
- `metadata.client_type = GAS_DEMO`
- `metadata.demo_version = demo_v1`

If DEMO header is missing/invalid:

- run fails as invalid demo mode response
- engineering blocks remain empty
- transport/error block is displayed

## Error Handling Implemented

API error envelope path:

- reads and displays `error_code`, `message`, `source_field`, `notes`, `request_id`
- does not write engineering blocks

Transport path (timeout/unavailable/non-JSON):

- writes `TRANSPORT_ERROR` block with message and timestamp
- does not write engineering blocks
